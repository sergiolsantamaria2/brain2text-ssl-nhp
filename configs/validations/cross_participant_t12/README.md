# Phase A — Cross-participant SSL generalization (T12 / B2T24)

**Goal:** validate that the winning SSL pretraining (AR binary, somatosensory
NHP 10h, epoch 400 — our best checkpoint on T15, PER=0.0887) also helps when
finetuning on a *different* human participant: T12 from the Brain-to-Text '24
competition (Willett 2023, Dryad Sep 2023 release).

**Design:** two config families × three seeds each = 6 runs.

| Family | SSL checkpoint | Seeds |
|--------|----------------|-------|
| `ft_t12_ssl_ar_binary_soma/` | `ar_binary_soma/epoch_0400.pt` | 10, 42, 123 |
| `ft_t12_no_ssl/` (control)   | none | 10, 42, 123 |

All other hyperparameters match the winning ssl_v2 finetuning recipe
(resffn head, lr_max=3e-4, wd=5e-4, cosine 200k, white_noise_std=0.8,
constant_offset_std=0.2, log_transform=true, epsilon=1e-8).

The dataset block is loaded from `configs/experiments/06_phase_a_t12/dataset_t12.yaml`,
which is auto-generated by `scripts/preprocess_b2t24.py --write-dataset-config …`
and lists every T12 session that ends up with a non-empty `data_train.hdf5`.

**Launch:** `sbatch scripts/run_phase_a_t12.sbatch` — uses `--gres=gpu:2` and
runs the SSL family on GPU 0 and the control family on GPU 1 in parallel.

**W&B group:** `phase_a_t12_generalization`. Tags: `phase_a`, `t12`, `b2t24`,
`generalization`.
